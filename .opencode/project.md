\# Cleaner Corecon.us



\## Architecture



\- Follow clean architecture principles

\- Keep components modular

\- Prefer composition over inheritance

\- Avoid business logic inside UI



\## Frontend



\- Use ui-ux-pro-max for UI reasoning and design systems

\- Follow DESIGN.md as primary visual reference

\- Use references/linear.md for UX interaction patterns

\- Use references/claude.md for AI/chat interfaces



\## UI Constraints



\- Avoid excessive animations

\- Avoid unnecessary visual effects

\- Avoid deep component nesting

\- Prefer semantic HTML

\- Prefer accessibility-first patterns

\- Mobile-first responsive design

\- Keep layouts clean and minimal



\## Stack



\- Next.js

\- TypeScript

\- Tailwind

\- shadcn/ui



\## Code Quality



\- Strict TypeScript

\- Prefer readability over cleverness

\- Small reusable components

\- Avoid overengineering



\## Security



\- Never expose secrets

\- Validate external input

\- Sanitize outputs
